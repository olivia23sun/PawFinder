# RWD 響應式設計規範 (Responsive Design)

本專案採用 **`rem` 單位系統** 搭配 **根字體縮放策略**，確保網站在不同裝置下皆能維持視覺比例的一致性。

## 1. 斷點設定 (Breakpoints)

我們根據市場主流裝置解析度，定義以下斷點作為樣式切換基準：

| 裝置類型 | 斷點 (Max-width) | 說明 |
| --- | --- | --- |
| **Desktop** | 1025px+ | 桌機與大螢幕筆電 |
| **Tablet** | 1024px | 標準平板 (iPad Pro 等) |
| **Small Tablet** | 768px | 直向平板 (iPad) |
| **Mobile** | 480px | 大尺寸手機 |
| **Small Mobile** | 360px | 小尺寸手機 (iPhone SE 等) |

---

## 2. 字體縮放策略

透過調整 HTML 根節點的 `font-size`，實現全站比例自動連動。

### 實際運作效果 (以 `.title { font-size: 2rem; }` 為例)

* **桌機 (1920px+)：**  (Base: 18px)
* **筆電 (1440px)：**  (Base: 17px)
* **平板 (768px)：**  (Base: 15px)
* **手機 (480px)：**  (Base: 13px)
* **小手機 (360px)：**  (Base: 12px)

---

## 3. 建議使用規格 (Reference Guide)

為維持排版一致性，請參考以下建議的 `rem` 數值：

* **大標題 (H1/Hero)：** `2rem` ~ `3rem`
* **標準內文 (Body)：** `1rem` (預設值)
* **次要內文 / 註記：** `0.875rem`
* **按鈕 (Button)：** `1rem`
* **標籤 (Tag / Caption)：** `0.75rem` ~ `0.875rem`

---

## 4. 核心優點

1. **全局縮放：** 僅需透過 CSS Media Query 調整 `html { font-size: ... }`，全站元素隨之連動。
2. **比例一致：** 避免在不同螢幕尺寸下，標題過大或內文過小的視覺衝突。
3. **開發效率：** 無須針對每個 class 重複編寫 `@media` 規則，減少程式碼冗餘。
4. **維護性高：** 調整設計系統時，只需修改單一變數即可完成全站更新。

---

### 優化重點說明：

* **加入表格：** 斷點設定改用表格呈現，一眼就能對應裝置與數值。
* **數學公式化：** 使用 LaTeX 渲染計算式（例如 ），看起來更專業且清晰。
* **增加層次：** 使用數字編號與水平線 (`---`) 分隔區域，提升閱讀體驗。
* **重點標註：** 適度使用 **粗體** 導引讀者目光。
